# Quickstart Guide

This guide walks you through making your first API calls to the E-Commerce API. You'll learn how to register a user, authenticate, browse products, manage a shopping cart, and create orders.

## Base URL

All API requests should be made to:

```
http://localhost:3000
```

## Authentication

Most endpoints require authentication using a JWT token. After logging in, include the token in the `Authorization` header:

```
Authorization: Bearer YOUR_JWT_TOKEN
```

---

<Steps>

<Step title="Register a New User">

Create a new customer account by sending a POST request to `/auth/register`.

<CodeGroup>

```bash cURL
curl -X POST http://localhost:3000/auth/register \
  -H "Content-Type: application/json" \
  -d '{
    "name": "John Doe",
    "email": "john@example.com",
    "password": "securepassword123"
  }'
```

```javascript JavaScript
const response = await fetch('http://localhost:3000/auth/register', {
  method: 'POST',
  headers: {
    'Content-Type': 'application/json',
  },
  body: JSON.stringify({
    name: 'John Doe',
    email: 'john@example.com',
    password: 'securepassword123'
  })
});

const data = await response.json();
console.log(data);
```

```python Python
import requests

response = requests.post(
    'http://localhost:3000/auth/register',
    json={
        'name': 'John Doe',
        'email': 'john@example.com',
        'password': 'securepassword123'
    }
)

print(response.json())
```

</CodeGroup>

**Response:**

```json
{
  "message": "Usuario registrado exitosamente",
  "user": {
    "id": "1",
    "name": "John Doe",
    "email": "john@example.com",
    "role": "customer",
    "createdAt": "2026-03-06T10:30:00.000Z"
  }
}
```

<Note>
Passwords must be at least 6 characters long. The password is hashed and never returned in API responses.
</Note>

</Step>

<Step title="Login to Get Access Token">

Authenticate with your credentials to receive a JWT token for subsequent requests.

<CodeGroup>

```bash cURL
curl -X POST http://localhost:3000/auth/login \
  -H "Content-Type: application/json" \
  -d '{
    "email": "john@example.com",
    "password": "securepassword123"
  }'
```

```javascript JavaScript
const response = await fetch('http://localhost:3000/auth/login', {
  method: 'POST',
  headers: {
    'Content-Type': 'application/json',
  },
  body: JSON.stringify({
    email: 'john@example.com',
    password: 'securepassword123'
  })
});

const data = await response.json();
const token = data.token;
console.log('Token:', token);
```

```python Python
import requests

response = requests.post(
    'http://localhost:3000/auth/login',
    json={
        'email': 'john@example.com',
        'password': 'securepassword123'
    }
)

data = response.json()
token = data['token']
print('Token:', token)
```

</CodeGroup>

**Response:**

```json
{
  "token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
  "user": {
    "id": "1",
    "name": "John Doe",
    "email": "john@example.com",
    "role": "customer"
  }
}
```

<Warning>
Store the JWT token securely. You'll need it for all authenticated requests. The token should be included in the `Authorization` header as `Bearer YOUR_TOKEN`.
</Warning>

</Step>

<Step title="Add Products to Cart">

Add items to your shopping cart. This endpoint requires authentication.

<CodeGroup>

```bash cURL
curl -X POST http://localhost:3000/cart/add \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer YOUR_JWT_TOKEN" \
  -d '{
    "productId": 1,
    "quantity": 2
  }'
```

```javascript JavaScript
const response = await fetch('http://localhost:3000/cart/add', {
  method: 'POST',
  headers: {
    'Content-Type': 'application/json',
    'Authorization': `Bearer ${token}`
  },
  body: JSON.stringify({
    productId: 1,
    quantity: 2
  })
});

const cartItem = await response.json();
console.log(cartItem);
```

```python Python
import requests

response = requests.post(
    'http://localhost:3000/cart/add',
    headers={'Authorization': f'Bearer {token}'},
    json={
        'productId': 1,
        'quantity': 2
    }
)

cart_item = response.json()
print(cart_item)
```

</CodeGroup>

**Response:**

```json
{
  "id": 1,
  "userId": 1,
  "productId": 1,
  "quantity": 2,
  "createdAt": "2026-03-06T11:00:00.000Z"
}
```

<Note>
The quantity must be at least 1. If you add the same product again, the quantities will be combined.
</Note>

</Step>

## Admin Operations

Admin users have additional privileges to manage products and categories. To perform admin operations, you must first create an admin user.

### Create Admin User

Only existing admins can create new users with specific roles:

<CodeGroup>

```bash cURL
curl -X POST http://localhost:3000/auth/create-user \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer ADMIN_JWT_TOKEN" \
  -d '{
    "name": "Admin User",
    "email": "admin@example.com",
    "password": "securepassword123",
    "role": "admin"
  }'
```

```javascript JavaScript
const response = await fetch('http://localhost:3000/auth/create-user', {
  method: 'POST',
  headers: {
    'Content-Type': 'application/json',
    'Authorization': `Bearer ${adminToken}`
  },
  body: JSON.stringify({
    name: 'Admin User',
    email: 'admin@example.com',
    password: 'securepassword123',
    role: 'admin'
  })
});

const admin = await response.json();
console.log(admin);
```

```python Python
import requests

response = requests.post(
    'http://localhost:3000/auth/create-user',
    headers={'Authorization': f'Bearer {admin_token}'},
    json={
        'name': 'Admin User',
        'email': 'admin@example.com',
        'password': 'securepassword123',
        'role': 'admin'
    }
)

admin = response.json()
print(admin)
```

</CodeGroup>

### Create a Product

Admin users can create new products with image uploads using multipart/form-data:

<CodeGroup>

```bash cURL
curl -X POST http://localhost:3000/products \
  -H "Authorization: Bearer ADMIN_JWT_TOKEN" \
  -F "name=Wireless Mouse" \
  -F "description=Ergonomic wireless mouse with precision tracking" \
  -F "price=49.99" \
  -F "stock=100" \
  -F "categoryId=1" \
  -F "image=@/path/to/mouse.jpg"
```

```javascript JavaScript
const formData = new FormData();
formData.append('name', 'Wireless Mouse');
formData.append('description', 'Ergonomic wireless mouse with precision tracking');
formData.append('price', '49.99');
formData.append('stock', '100');
formData.append('categoryId', '1');
formData.append('image', fileInput.files[0]); // From file input element

const response = await fetch('http://localhost:3000/products', {
  method: 'POST',
  headers: {
    'Authorization': `Bearer ${adminToken}`
  },
  body: formData
});

const product = await response.json();
console.log('Product created:', product);
```

```python Python
import requests

files = {'image': open('/path/to/mouse.jpg', 'rb')}
data = {
    'name': 'Wireless Mouse',
    'description': 'Ergonomic wireless mouse with precision tracking',
    'price': 49.99,
    'stock': 100,
    'categoryId': 1
}

response = requests.post(
    'http://localhost:3000/products',
    headers={'Authorization': f'Bearer {admin_token}'},
    files=files,
    data=data
)

product = response.json()
print('Product created:', product)
```

</CodeGroup>

**Response:**

```json
{
  "id": 3,
  "name": "Wireless Mouse",
  "description": "Ergonomic wireless mouse with precision tracking",
  "price": 49.99,
  "stock": 100,
  "imageUrl": "http://localhost:3000/uploads/wireless-mouse-1234567890.jpg",
  "categoryId": 1,
  "createdAt": "2026-03-06T12:00:00.000Z"
}
```

<Note>
Product images are required and must be less than 5MB. Supported formats include JPEG, PNG, and WebP.
</Note>

## Common Issues

<AccordionGroup>
  <Accordion title="401 Unauthorized Error">
    This error occurs when your JWT token is missing, invalid, or expired. Make sure you're including the token in the `Authorization` header as `Bearer YOUR_TOKEN`. You may need to login again to get a fresh token.
  </Accordion>
  
  <Accordion title="Rate Limit Exceeded">
    Authentication endpoints (`/auth/register` and `/auth/login`) are rate limited to 10 requests per 15 minutes. If you exceed this limit, wait 15 minutes before trying again.
  </Accordion>
  
  <Accordion title="Product Image Upload Fails">
    Ensure your image is less than 5MB and in a supported format (JPEG, PNG, WebP). The image field must be named `image` in the multipart form data.
  </Accordion>
  
  <Accordion title="Cannot Create Order from Empty Cart">
    You must have at least one item in your cart before creating an order. Add products to your cart using `/cart/add` first.
  </Accordion>
</AccordionGroup>
